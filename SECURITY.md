# Security Policy — Old World Builder

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x | ✅ Active development |
| < 1.0 | ❌ No longer supported |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately via the project's GitHub Issues or Discord server.

1. **DO NOT** file a public issue on GitHub.
2. Open a private advisory on GitHub or contact via Discord with:
   - A description of the vulnerability
   - Steps to reproduce
   - Affected versions
   - Potential impact
3. You should receive a response within 48 hours.

## Disclosure Policy

- Vulnerabilities will be acknowledged within 48 hours.
- A fix will be developed and tested.
- Once a fix is released, a public advisory will be published.
- Reporters will be credited (unless they request anonymity).

## Security Measures

- All dependencies are regularly audited.
- Code reviews are required for all changes.
- Access to production systems is restricted and logged.
